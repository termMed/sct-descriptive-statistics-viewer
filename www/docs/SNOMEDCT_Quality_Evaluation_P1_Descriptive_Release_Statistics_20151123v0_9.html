1 Introduction

1.1 Background

 

This document describes general descriptive release statistics and additional queries to be used as part of an overall Quality Assurance process for the SNOMED CT International Release for which the objective is to provide information to assist the IHTSDO in:

 

  • Ensuring that newly added content meets quality assurance criteria
  • Ensuring that changes to existing content meet quality assurance criteria
  • Identifying and prioritizing areas of existing content which do not currently meet quality assurance criteria
  • Providing metrics on temporal trends as a measurement of progress

1.2 Purpose

 

This purpose of this document is to propose an initial set of descriptive measures and statistics, obtained through automated queries, with the objective of providing information on content changes and temporal trends as a measurement of progress. The gathered data will also assist in comparing consecutive SNOMED CT¨ International Releases and trends across multiple releases.

 

1.3 Scope

 

This document describes:

á       A set of proposed measures and statistics to be computed from SNOMED CT releases

A complementary document will describe a set of content patterns recommended for periodic monitoring and evaluation of conformance with content management, quality assurance processes and technical specifications.

1.4 Audience

 

The audience for this document includes IHTSDO Content staff and the Community of Practice.

3 Descriptive statistics on new concepts

3.1 Background

Factors affecting the number of new Concepts in the release include but are not limited to;

  1. Ongoing content projects – influenced by:
    1. Prioritization
    2. Policy development
    3. Project timelines
  2. Request Submissions (single and batch) – influenced by:
    1. Submission timing with respect to the release cycle
    2. Submission appropriateness for the International Release
    3. Submission complexity
    4. Submission quality
    5. State of existing content in the hierarchy where the submission is added
    6. Ongoing changes to editorial policy

New concepts can be added with a definition status of sufficiently defined (SD) or primitive (P). A concept is defined by a set of relationships to other concepts. If the resulting definition is sufficient to distinguish the concept from its parents and siblings in the subtype hierarchy, the concept is considered to be sufficiently defined. If the definition is not sufficient to distinguish the concept from its parents and siblings, the concept is considered to be primitive (SNOMED CT¨ Technical Implementation Guide January 2015 International Release Section 4.2.1.2).

Factors affecting the definition status of new concepts include but are not limited to:

  1. The Concept Model does not allow sufficiently defined concepts in some hierarchies.
  2. For hierarchies, which allow sufficiently defined concepts, the Concept Model may not be expressive enough to support sufficiently defining some concepts.
  3. Some concepts may be intrinsically primitive and unable to be sufficiently defined (within the clinical scope of SNOMED CT) even with enhancements to the Concept Model.

3.2 Statistics for new concepts as part of the overall release

¤  Number of new Concepts

¤  Percentage of new Concepts which are sufficiently defined

4.2 Statistics on existing concepts (active in release 1 and in release 2)

The statistics provided for changes to existing Concepts (active Concepts in R1 which remain active in R2) for the entire release and for each hierarchy include:

¤  Number and percentage of Concepts that underwent any change

¤  Number and percentage of Concepts with any stated change to the concept definition

¤  Number and percentage of Concepts with any inferred change to the concept definition

¤  Number and percentage of Concepts with a definition status change from P to SD

¤  Number and percentage of Concepts with a definition status change from SD to P

¤  Number and percentage of Concepts with any Description change

¤  Number and percentage of Concepts with a FSN change

Details for each query are provided in the sections below.

4.2.1 Number and percentage of Concepts that underwent any change

Statistics are provided on the number and percentage of Concepts that underwent any change, which includes any of the following:

¤  Change to the concept definition

Includes:

o   Change to supertype Relationships ( | Is a | Relationships to immediate parents)

o   Change to defining Relationships (defining attribute-value pairs)

 

¤  Change to the concept definition status

Includes:

    • Change from SD to P
    • Change from P to SD

¤  Change to Descriptions

Includes:

    • Adding new Descriptions
    • Retiring Descriptions

¤  Change to the FSN

4.2.2 Number and percentage of Concepts with any change to the concept definition

 

Statistics are provided on:

¤  Number and percentage of Concepts with any stated change to the concept definition

¤  Number and percentage of Concepts with any inferred change to the concept definition

Change to a concept definition includes any of the following:

¤  Change to | Is a | Relationships (of immediate parent Concepts)

Includes:

    • New | Is a | Relationships as a result of:
      • New stated | Is a | Relationships (manual edits)
      • New inferred | Is a | Relationships as a result of:
        • Change to the concept definition
        • Change to the new parentÕs concept definition
        • Change to the new parentÕs concept definition status

 

    • Inactivated | Is a | Relationships as a result of:
      • Inactivated stated | Is a | Relationships (manual edits)
      • Inactivated inferred | Is a | Relationships as a result:
        • Change to the concept definition
        • Change to the parentÕs concept definition
        • Change to the parentÕs concept definition status

 

  • Change to defining Relationships (attribute-value pairs)

Includes:

    • New attribute-value pairs as a result of:
      • New stated attribute-value pairs (manual edits)
      • New inferred attribute-value pairs as a result of:
        • Change to stated attribute-value pairs of one or more of the ConceptÕs stated ancestors
        • New stated parent Concepts being added to the ConceptÕs ancestry
        • Change to the relationship between the value used for an attribute in the definition of the Concept and a value used for the same attribute in the concept definition of an ancestor (e.g., the value for the attribute in the ConceptÕs definition was, but is no longer, a subtype of a value used for the same attribute in a stated ancestor.)

In this scenario, the attribute-value pair of the ancestor Concept is no longer more general than the attribute-value pair of the descendant Concept and is no longer removed when the Distribution Normal Form (DNF) is computed. Instead, the attribute-value pair of the ancestor Concept is now inherited (inferred) in the concept definition of the descendant Concept. (While inferred changes to a ConceptÕs definition might be part of normal editing processes, this category of change might also be the side effect of an anomaly that requires revision of the ConceptÕs definition.)

 

    • Retired attribute-value pairs as a result of:
      • Retired stated attribute-value pairs in the ConceptÕs definition (manual edits)
      • Retired inferred attribute-value pairs in the ConceptÕs definition as a result of:
        • Changes to a stated definition of one or more of the ConceptÕs stated ancestors
        • Removal of one of ConceptÕs stated ancestors
        • Change to the relationship between the value used for an attribute in the definition of the Concept and a value used for the same attribute in the concept definition of an ancestor (e.g., the value for the attribute in the ConceptÕs definition was not previously, but now is, a subtype of a value used for the same attribute in a stated ancestor.)

In this scenario, the attribute-value pair of the ancestor Concept is now more general than the attribute-value pair of the descendant Concept and is removed when the Distribution Normal Form is computed. The attribute-value pair of the ancestor Concept is now no longer inherited (inferred) in the concept definition of the descendant Concept. Unlike its counterpart described under ÒNew inferred attribute-value pairsÓ above, this type of change may represent a deliberate change in the relationship between the values, which was made to improve/correct the ConceptÕs definition.

Number and percentage of Concepts with a stated change to the concept definition

Stated changes are changes that made directly by the editors to the set of Relationships to other Concepts that make up the conceptÕs Description Logic definition. They do not include Òinferred changesÓ to the conceptÕs definition which are the result of inferences about the relationships between sets of concepts after they are processed by a description logic classifier (autoclassification).

Statistics are provided on the number and percentage of Concepts which underwent a stated change to the concept definition which includes any of the following:

  • Change to stated | Is_a | Relationships

Includes:

    • New stated | Is a | Relationships
    • Retired stated | Is a | Relationships
  • Change to stated defining Relationships (attribute-value pairs)

Includes:

    • New stated attribute-value pairs
    • Retired stated attribute-value pairs

Number and percentage of Concepts with an inferred change to the concept definition

Inferred changes are not made directly by the editors. Inferred changes to the conceptÕs definition are generated as result of inferences about the stated relationships between sets of concepts after they are processed by a description logic classifier (autoclassification). These are changes to the inferred Relationships distributed in the main Relationships files of the official distribution of SNOMED CT.

Statistics are provided on the number and percentage of Concepts that underwent an inferred change to the concept definition, which includes any of the following:

  • Change to inferred | Is_a | Relationships

Includes:

    • New inferred | Is a | Relationships
    • Retired inferred | Is a | Relationships
  • Change to inferred defining Relationships (attribute-value pairs)

Includes:

    • New inferred attribute-value pairs
    • Retired inferred attribute-value pairs

4.2.3 Number and percentage of Concepts with a definition status change

A ConceptÕs definition status can be primitive or sufficiently defined. A Concept is defined by a set of relationships to other Concepts. If the resulting definition is sufficient to distinguish the Concept from its parents and siblings in the subtype hierarchy, the concept is considered to be sufficiently defined. If the definition is not sufficient to distinguish the concept from its parents and siblings, the concept is considered to be primitive. (SNOMED CT¨ Technical Implementation Guide January 2015 International Release Section 4.2.1.2)

Statistics are provided on following changes to definition status:

  • Number and percentage of Concepts with a definition status change from P to SD
  • Number and percentage of Concepts with a definition status change from SD to P

Number and percentage of Concepts with a definition status change from P to SD

Reasons for changing a ConceptÕs definition status from P to SD include any of the following:

  • The concept definition is changed to sufficiently define the concept as a result of:
    • Adding new Relationships to other concepts (existing concepts or new concepts in the release)
    • Changes to the Concept Model (e.g., new attributes or changes to Editorial Guidance for existing attributes)
    • Removing Relationships to existing concepts (which were incorrect)
  • The conceptÕs existing definition is reviewed and felt to be adequate to sufficiently define the Concept

Number and percentage of Concepts with a definition status change from SD to P

Reasons for changing a ConceptÕs definition status from SD to P include any of the following:

  • The concept definition is changed and the concept is no longer sufficiently defined as a result of:
    • Removing Relationships to other concepts (e.g., existing concepts which have been retired in the release)
    • Changes to the Concept Model (e.g., changes to Editorial Guidance for attributes)
  • The conceptÕs existing definition is reviewed and felt to be inadequate to sufficiently define the Concept

4.2.4 Number and percentage of Concepts with any Description change

Descriptions include FSNs, Preferred terms, and/or Synonyms.

Statistics are provided on the number and percentage of Concepts that underwent any Description change which includes any of the following:

  • Retiring existing Descriptions (e.g., Descriptions which are inappropriate, outdated, spelling errors etc.)
  • Adding new Descriptions

4.2.5 Number and percentage of Concepts with a Fully Specified Name (FSN) change

Statistics are provided on the number and percentage of Concepts which underwent a FSN change.

Changes to FSNs also contribute to the total number of Description changes described above. However, statistics on the changes to FSNs are also provided separately because the FSN represents the meaning of the Concept code and policy regarding changes to FSNs is stricter than policy regarding changes to Synonyms or Preferred Terms. A change to an FSN requires retiring the concept except in cases where the Fully Specified Name (FSN) undergoes a minor change. Minor changes are allowed to existing FSNs without retiring the concept. Per the July 2015 Editorial Guide Section 6.2.4.3.1:

ÒMinor changes in the FSN are those changes that do not alter its meaning. They may include changed capitalization, punctuation, spelling, and acronym expansion or word order revision. Such changes are sometimes necessary in order to achieve a consistent and predictable presentation style, but are allowed only if they do not change the Concept's meaning.
